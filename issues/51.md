## UI / CSS

### Goals
- Sleek
- Functional

### Stack
- Tailwind CSS
- daisyUI

### Design Rules
- Layout: Provide a clean and intuitive layout that allows users to easily navigate through the application.
- Cards/Tables: Use cards for displaying grouped information and tables for data representation where applicable.
- Typography: Ensure readability with appropriate font sizes and weights.
- Light/Dark Mode: Implement light and dark themes to cater to different user preferences.

### SSE Status Indicator
- Include a status indicator in the header to show the current state of SSE connections.

### Page-specific Notes
- **Dashboard**: Focus on quick insights and actionable items.
- **Runs**: Track and display the results of user runs.
- **Logs Viewer**: Provide a searchable and filterable log view.
- **Plan Diff (Responsive)**: Ensure the plan diff is easily readable and navigable on all device sizes.
- **Managed Units (Read-Only)**: Display managed units in a read-only format.

### Accessibility Basics
- Use ARIA roles and properties where necessary to enhance accessibility.
- Ensure keyboard navigability for all interactive elements.

### Additional UI Acceptance Criteria
- All UI elements should be responsive and retain functionality across different screen sizes.
- Colors and contrasts should meet accessibility standards for visibility.
- The application should provide consistent spacing and margins across similar components.
